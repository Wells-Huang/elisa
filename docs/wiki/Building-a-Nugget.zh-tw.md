# 建置 Nugget (Building a Nugget)

當您使用積木設計好專案後，按下 GO 按鈕即可開始建置。

## GO 按鈕

GO 按鈕位於標題列的右上角。

| 狀態 | 外觀 | 代表意義 |
|-------|-----------|---------|
| **準備就緒 (Ready)** | 綠色，輕微閃爍 | 您的設計包含目標且後端已就緒。點擊即可開始建置。 |
| **停用 (Disabled)** | 變暗 | 缺少必要內容：未放置目標積木，或後端尚未就緒。 |
| **建置中 (Building)** | 顯示「STOP」 | 正在建置中。點擊「STOP」可取消建置。 |

當滿足以下三個條件時，GO 按鈕即可使用：
- 您處於設計模式（尚未開始建置）
- 您的工作區有一個包含文字的「目標」積木
- 就緒勳章顯示「就緒 (Ready)」

## 選擇工作區資料夾

第一次按下 GO 時，Elisa 會要求您選擇一個資料夾來儲存產出的專案檔案。在 Electron 應用程式中，這會開啟系統的原生資料夾選取器。在開發模式下，則需手動輸入路徑。

此資料夾會被記住供日後使用。您可以透過側欄的「資料夾」按鈕更改路徑。

## 建置生命週期

建置會經歷以下階段：

```
設計 (design) → 建置 (building) → 檢閱 (review, 選填) → 部署 (deploy) → 完成 (done)
```

| 階段 | 發生了什麼事 |
|-------|-------------|
| **設計 (design)** | 您正在畫布上編輯積木。 |
| **建置 (building)** | AI 代理正在規劃任務、撰寫程式碼並執行測試。視圖會切換至「任務控制中心」。 |
| **檢閱 (review)** | 因為觸發了「與我確認」積木，建置暫停。您批准或拒絕。 |
| **部署 (deploy)** | 程式碼已部署（網頁預覽、硬體燒錄或兩者皆有）。 |
| **完成 (done)** | 建置完成。出現摘要對話框。 |

### 技術細節

1. 積木解釋器將您的工作區轉換為 **NuggetSpec** JSON（透過 Zod 結構驗證）。
2. NuggetSpec 透過 `POST /api/sessions/:id/start` 傳送至後端。
3. **元規劃器 MetaPlanner** (Claude Opus) 將規格分解為 **任務 DAG** —— 一個具備相依關係的有向無環圖任務集合。
4. **協調器 Orchestrator** 使用串流並行執行池執行任務（透過 Promise.race，最多 3 位並行代理）。
5. 每個代理透過 Claude Agent SDK 的 `query()` API 以角色特定提示詞運行。
6. 完成每項任務後，會建立 Git 提交，並為後續代理寫下上下文摘要。
7. 測試透過 pytest (Python) 或 Node 測試執行器 (JavaScript) 運行。
8. 最後執行部署：網頁預覽、ESP32 燒錄或門戶執行。

## 任務控制中心 (Mission Control)

建置開始後，畫面會切換到「任務控制中心」分頁。它包含三個區塊：

### 任務圖 (Task DAG，左側)
一個互動式圖表，顯示所有任務節點與相依箭頭。節點顏色代表狀態：
- **灰色** = 待處理
- **藍色** = 進行中
- **綠色** = 已完成
- **紅色** = 失敗

### 僕從小隊 (Minion Squad，右上)
顯示每位代理的卡片，包含其名稱、角色、個性及目前狀態（閒置、工作中、完成、錯誤）。

### 講述者饋送 (Narrator Feed，右下)
滾動式的講述者訊息，以適合兒童的語言描述正在發生的事。講述者有不同的情緒：興奮、鼓勵、擔憂、慶祝。

## 底部標籤欄

底部欄有六個標籤，會在建置過程中填滿資訊：

| 標籤 | 顯示內容 |
|-----|--------------|
| **時間軸 (Timeline)** | 代理所做的 Git 提交紀錄與已變更檔案清單 |
| **測試 (Tests)** | 測試結果（通過/失敗）與程式碼覆蓋率百分比 |
| **開發板 (Board)** | 已連接 ESP32 開發板的序列埠輸出 |
| **學習 (Learn)** | 學習時刻 —— 程式設計概念的簡短解說 |
| **進度 (Progress)** | 建置進度條，顯示目前階段與任務完成狀況 |
| **Token** | 每位代理的 Token 使用量、總成本及預算百分比 |

## 停止建置

建置期間，GO 按鈕會變成 STOP。按下即可取消。所有代理會透過 AbortController 信號傳播停止工作，您將返回「完成」狀態。

## 人類閘門 (Human Gates)

若您放置了「與我確認」積木，建置會暫停，並顯示包含後續進度的彈窗。您可以：
- **批准 (Approve)** 以繼續建置
- **拒絕 (Reject)** 以停止

## 代理提問

有時候代理在建置過程中需要您的輸入。此時會出現提問彈窗。您的回答將引導代理下一步的行動。

## 「繼續工作」流程

建置完成時，「Nugget 建置完成！」對話框提供三個選項：

- **在瀏覽器開啟** — 若專案已部署到網頁，會在分頁開啟。
- **建置新項目** — 重新整理頁面以進行全新的開始。
- **繼續改進這個 Nugget** — 返回設計模式。下次建置會從上次停下的地方開始，保留現有的程式碼與 Git 歷程。

## Token 預算

每個建置工作階段預設有 500,000 Token 的預算。當使用量達到 80% 時會出現警告。當超出預算時，建置會優雅停止。詳見[疑難排解 (Troubleshooting)](Troubleshooting) 以獲取在預算內工作的提示。
