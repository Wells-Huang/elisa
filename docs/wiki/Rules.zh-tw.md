# 規則 (Rules)

規則是告訴您的 AI 代理「必須做什麼」或「不能做什麼」的護欄。與增加能力的 [技能 (Skills)](Skills.zh-tw.md) 不同，規則的作用是增加限制（約束）。

## 規則的作用

規則會在建置過程中的特定時間點自動觸發。例如，觸發條件為「部署前」的規則會在專案部署前一刻進行檢查 —— 您不需要手動執行它。

透過將「套用規則」[積木 (Block-Reference#rules)](Block-Reference.zh-tw.md#規則-rules) 拖曳到畫布上，即可將規則附加到您的專案。

## 建立規則

1. 點擊側欄的 **規則 (Rules)** 按鈕。
2. 點擊 **+ 新增規則 (New Rule)**。
3. 輸入 **名稱**（例如：「務必加上註釋」）。
4. 選擇 **何時套用**（觸發條件）：
   - **總是開啟 (Always on)** — 規則適用於整個建置過程。
   - **任務完成時 (On task complete)** — 當代理完成一項任務時進行檢查。
   - **測試失敗時 (On test fail)** — 當測試失敗時套用。
   - **部署前 (Before deploy)** — 在部署前一刻進行檢查。
5. 撰寫 **規則說明** — 代理必須執行或檢查的事項。
6. 點擊 **完成 (Done)**。

## 編輯與刪除規則

在規則彈窗中，每個規則都會顯示其名稱、觸發條件標籤以及說明的預覽。點擊 **編輯 (Edit)** 進行修改，或點擊 **刪除 (Delete)** 將其移除。

## 範本庫

點擊 **從範本建立 (From Template)** 來瀏覽預設規則：

| 範本名稱 | 觸發條件 | 作用 |
|----------|---------|-------------|
| 務必加上註釋 | 總是 | 使用淺顯易懂的語言為每個函式標註註釋 |
| 測試每項功能 | 總是 | 每項功能至少包含一個測試案例 |
| 移除 console.log | 任務完成時 | 在結束前移除所有除錯用的日誌 |
| 檢查失效連結 | 任務完成時 | 驗證所有連結與圖片來源是否正常運作 |
| 一次只修補一件事 | 測試失敗時 | 專注於最小化的修復，不進行重構 |
| 先閱讀錯誤訊息 | 測試失敗時 | 在修改程式碼前先分析錯誤訊息 |
| 必須編譯成功 | 部署前 | 部署前不得有任何錯誤或警告 |
| 所有測試必須通過 | 部署前 | 部署前所有測試套件必須全數通過 |

點擊 **新增 (Add)** 即可將範本加入您的規則庫中。

## 規則如何影響代理

當規則觸發時，其說明會被加入到該任務的代理提示詞中。代理會將該規則視為必須遵循的需求。

- **總是 (Always)** 規則會包含在每位代理的提示詞中。
- **任務完成時 (On task complete)** 規則會在每項任務結束後進行檢查。
- **測試失敗時 (On test fail)** 規則會在代理需要修復失敗測試時提供指引。
- **部署前 (Before deploy)** 規則會在部署階段進行檢查。「部署前」規則會呈現為 `deploy_checklist` 事件。

> **試試看**：開啟規則彈窗，點擊「從範本建立」，並新增「務必加上註釋」規則。接著將一個「套用規則」積木拖到畫布上並從下拉選單選取該規則。您下次建置的程式碼中，每個函式都會包含詳細的註釋。
