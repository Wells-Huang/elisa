# 儲存與讀取 (Saving and Loading)

Elisa 會儲存您的專案設計（積木、技能、規則與門戶），以便您稍後可以返回繼續。

## 自動儲存到瀏覽器

每當您進行變更時，您的工作區都會自動儲存到瀏覽器的 `localStorage`。當您重新開啟 Elisa 時，所有內容都會被恢復。不需要手動儲存。

這包含：
- 畫布上的積木佈圖
- 所有 [技能 (Skills)](Skills.zh-tw.md)
- 所有 [規則 (Rules)](Rules.zh-tw.md)
- 所有 [門戶 (Portals)](Portals.zh-tw.md)
- 您選擇的工作區資料夾路徑

使用的 `localStorage` 鍵值：`elisa:workspace`, `elisa:skills`, `elisa:rules`, `elisa:portals`, `elisa:workspace-path`。

## 儲存到工作區資料夾

若您已設定工作區資料夾（透過側欄的「資料夾」按鈕或第一次按下 GO 時設定），點擊 **儲存 (Save)** 會將您的設計檔案直接寫入該資料夾：

- `workspace.json` — 您的積木佈圖
- `skills.json` — 您的技能
- `rules.json` — 您的規則
- `portals.json` — 您的門戶

工作區資料夾也是 Elisa 在建置過程中存放產出的專案程式碼的地方。

## .elisa 檔案格式

若未設定工作區資料夾，點擊 **儲存 (Save)** 會下載一個 `.elisa` 檔案。這是一個 zip 壓縮檔，包含：

```
workspace.json     — 積木佈圖
skills.json        — 技能
rules.json         — 規則
portals.json       — 門戶
output/            — 產出的程式碼 (若建置已完成)
```

您可以將副檔名更改為 `.zip` 並使用任何解壓縮工具開啟以檢視其內容。

## 開啟 .elisa 檔案

點擊側欄的 **開啟 (Open)** 並選取一個 `.elisa` 檔案。這會恢復：
- 畫布上的積木佈圖
- 所有技能、規則與門戶
- 產出的程式碼（若壓縮檔內包含這些內容）

## 從工作區資料夾讀取

點擊側欄的 **資料夾 (Folder)** 並選擇一個目錄。若該資料夾包含 `workspace.json`, `skills.json`, `rules.json` 或 `portals.json`，它們將會被載入。該資料夾也會成為您未來儲存與建置的活動工作區。

## Nugget 範例

Elisa 隨附了內建的範例專案，您可以從「範例選擇器 (Example Picker)」中載入：

| 範例專案 | 類別 | 建置內容 |
|---------|----------|---------------|
| 簡易網頁 App | 網頁 | 一個基礎的網頁應用程式 |
| 硬體閃爍 | 硬體 | 一個 ESP32 LED 閃爍程式 |
| 團隊建置 | 多代理 | 使用多位專業代理的專案 |
| 太空閃避 | 遊戲 | 太空風格的閃避遊戲 |
| 技能展示 | 網頁 | 透過功能豐富的建置展示技能用途 |
| 規則展示 | 網頁 | 透過注重質量的代理展示規則用途 |

「範例選擇器」會在首次啟動時（且不存在已儲存工作區時）自動開啟。您也可以隨時點擊側欄的 **範例 (Examples)** 按鈕來開啟它。

> **試試看**：點擊側欄的 **儲存** 來將目前的專案下載為 `.elisa` 檔案。關閉 Elisa 並重新開啟後，點擊 **開啟** 並選取您剛剛儲存的檔案。您的完整設計應該會完全恢復如初。
