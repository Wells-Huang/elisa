# 門戶 (Portals)

門戶將您的 Elisa 專案連接至外部工具、服務與硬體。它們讓您的 AI 代理能與外部世界互動。

## 機制類型

每個門戶使用以下四種機制之一：

| 機制 | 它是什麼 | 範例 |
|-----------|-----------|---------|
| **MCP 伺服器** | 提供工具的模組上下文協定 (Model Context Protocol) 伺服器 | 檔案系統 (File System)、GitHub、Brave Search |
| **CLI 工具** | 代理可以執行的命令行程式 | gcloud, curl, python3 |
| **序列埠 / USB** | 透過 USB 序列埠連接的硬體裝置 | ESP32 開發板、LoRa 無線電 |
| **自動偵測** | 由 Elisa 自動判斷最佳機制 | 通用型的備選方案 |

## 建立門戶

1. 點擊側欄的 **門戶 (Portals)** 按鈕。
2. 點擊 **+ 新增門戶 (New Portal)** 從頭開始，或點擊 **從範本建立 (From Template)** 以快速套用預設配置。
3. 填寫欄位：
   - **名稱** -- 此門戶的標籤（例如：「我的 ESP32 開發板」）
   - **說明** -- 連接對象的說明
   - **機制** -- 要使用的連線類型
4. 針對特定機制進行配置（詳見下文）。
5. 點擊 **完成 (Done)**。

### 序列埠 / USB 設定

- **序列埠** -- 連接埠名稱（例如：Windows 上的 `COM3`，Linux 上的 `/dev/ttyUSB0`）。留空可使用自動偵測。
- **波特率** -- 通訊速度（預設：115200）。

若目前已連接開發板，會顯示綠色橫幅指出偵測到的型號與連接埠。

### MCP 伺服器設定

- **指令** -- 要執行的程式（例如：`npx`）。
- **參數** -- 命令行參數（例如：`-y @anthropic-ai/mcp-filesystem`）。

### CLI 工具設定

- **指令** -- 要執行的程式（例如：`gcloud`）。

## 門戶範本

點擊 **從範本建立** 來瀏覽預先配置好的門戶：

| 範本 | 機制 | 能力 (Capabilities) |
|----------|-----------|-------------|
| **ESP32 開發板** | 序列埠 | LED 開/關/閃爍、讀取感測器、按鈕點擊、播放聲音 |
| **LoRa 無線電** | 序列埠 | 發送訊息、接收訊息 |
| **檔案系統** | MCP | 讀取檔案、寫入檔案、列出檔案 |
| **GitHub** | MCP | 建立議題、讀取儲存庫、搜尋程式碼 |
| **Brave 搜尋** | MCP | 網頁搜尋、在地搜尋 |
| **Cloud Run 部署** | CLI | 部署至 Cloud Run |

選擇範本將自動填入名稱、機制、配置以及能力。

## 能力類型

每個門戶都具備定義其功能的「能力」。共有三種：

| 種類 | 它是做什麼的 | 對應積木 |
|------|-------------|-------------|
| **告訴 (Tell)** (動作) | 向門戶發送指令 | 「告訴 ___ 執行 ___」 |
| **當...時 (When)** (事件) | 當某些事發生時做出反應 | 「當 ___ ___」 |
| **詢問 (Ask)** (查詢) | 向門戶請求資料 | 「詢問 ___ 關於 ___」 |

## 動態參數

某些能力具備參數，當您在積木上選取該能力時，會出現額外的欄位。參數類型包含文字、數字、布林值（核取方塊）以及選項（下拉選單）。

## 在設計中使用門戶

建立門戶後，從積木調色盤的「門戶」類別中，將門戶積木（告訴、當...時或詢問）拖曳到畫布上。第一個下拉選單選擇要使用的門戶，第二個則選擇該門戶的能力。

> **試試看**：點擊側欄的門戶按鈕，選擇「從範本建立」，然後選取「檔案系統」。儲存後，將「告訴」積木拖到畫布上。在下拉選單中選擇「檔案系統」與「寫入檔案」。現在，您的代理就能在建置過程中將檔案寫入硬碟了。
