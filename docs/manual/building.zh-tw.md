# 建置 (Building)

當您使用積木設計好專案後，按下 GO 按鈕即可開始建置。

## GO 按鈕

GO 按鈕位於標題列的右上角。它有幾種狀態：

| 狀態 | 外觀 | 代表意義 |
|-------|-----------|---------|
| **準備就緒 (Ready)** | 綠色，輕微閃爍 | 您的設計已包含「目標」且後端已就緒。點擊即可開始建置。 |
| **停用 (Disabled)** | 變暗 | 缺少必要內容：未放置「目標」積木，或後端尚未就緒。 |
| **建置中 (Building)** | 顯示「STOP」 | 正在建置中。點擊「STOP」可取消建置。 |

當滿足以下三個條件時，GO 按鈕即可使用：
- 您處於設計模式（尚未開始建置）
- 您的工作區有一個包含文字的「目標」積木
- 就緒勳章顯示「Ready」

## 選擇工作區資料夾

第一次按下 GO 時，Elisa 會要求您選擇一個資料夾來儲存產出的專案檔案。在 Electron 應用程式中，這會開啟系統的原生資料夾選取器；在開發模式下，則需手動輸入路徑。

此資料夾會被記住供日後使用。您可以透過側欄的「資料夾」按鈕更改路徑。

## 建置生命週期

建置會經歷以下階段：

```
設計 (design) --> 建置 (building) --> 檢閱 (review, 選填) --> 部署 (deploy) --> 完成 (done)
```

| 階段 | 發生了什麼事 |
|-------|-------------|
| **設計 (design)** | 您正在畫布上編輯積木。 |
| **建置 (building)** | AI 代理正在規劃任務、撰寫程式碼並執行測試。視圖會切換至「任務控制中心 (Mission Control)」。 |
| **檢閱 (review)** | 因為觸發了「與我確認」積木，建置暫停。您在彈窗中選擇批准或拒絕。 |
| **部署 (deploy)** | 程式碼已部署（網頁預覽、硬體燒錄或兩者皆有）。 |
| **完成 (done)** | 建置完成。出現摘要對話框，提供「建立新專案」或「繼續工作」的選項。 |

## 任務控制中心 (Mission Control)

建置開始後，畫面會切換到「任務控制中心」標籤。它包含三個區塊：

### 任務圖 (Task DAG，左側)
一個互動式圖表，將所有任務顯示為帶有相依關係箭頭的節點。節點顏色代表狀態：
- 灰色 = 待理處
- 藍色 = 進行中
- 綠色 = 已完成
- 紅色 = 失敗

### 僕從小隊 (Minion Squad，右上)
顯示每位代理的卡片，包含其名稱、角色、個性及目前狀態（閒置、工作中、完成、錯誤）。

### 講述者饋送 (Narrator Feed，右下)
一個滾動式的訊息饋送，講述者會以適合兒童的語言描述正在發生的事。講述者有不同的情緒狀態：興奮、鼓勵、擔憂、慶祝。

## 底部標籤欄

底部欄有六個標籤，會在建置過程中逐漸填滿資訊：

| 標籤 | 顯示內容 |
|-----|--------------|
| **時間軸 (Timeline)** | 代理所做的 Git 提交紀錄，包含變更檔案列表。每個提交都會註明是哪位代理為了哪個任務所做的變更。 |
| **測試 (Tests)** | 測試結果（通過/失敗）與程式碼覆蓋率百分比。測試階段會顯示進度指示器。 |
| **開發板 (Board)** | 已連接 ESP32 開發板的序列埠輸出。僅在開發板活動時顯示資料。 |
| **學習 (Learn)** | 學習時刻 —— 對於建置過程中出現的程式設計概念進行簡短解說。 |
| **進度 (Progress)** | 建置進度條，顯示目前階段與已完成任務的百分比。 |
| **Token** | 每個代理的 Token 使用量、總成本及預算百分比。顯示每位僕從的使用條。 |

建置開始時會自動選擇「進度」標籤。第一筆測試結果抵達時，會自動切換至「測試」標籤。

## 停止建置

建置期間，GO 按鈕會變成 STOP。按下即可取消建置。所有代理會停止工作，您將返回「完成」狀態，在那裡可以選擇重頭開始或繼續工作。

## 人類閘門 (Human Gates)

若您放置了「與我確認」積木，建置會暫停，並顯示一個包含問題及代理至今進度背景資訊的彈窗。您可以點擊「批准」以繼續，或點擊「拒絕」以停止。

## 代理提問

有時候代理在建置過程中需要您的輸入。此時會出現提問彈窗，供您選擇引導代理下一步該怎麼做的選項。您的答案將會指引代理的後續行動。

## 「繼續工作」流程

建置完成時，「Nugget 建置完成！」對話框提供三個選項：

- **在瀏覽器開啟** -- 若專案已部署到網頁，會在分頁開啟。
- **建立新專案** -- 重新整理頁面以進行全新的開始。
- **繼續改進這個專案** -- 返回設計模式並保留既有積木。下次建置會從上次停下的地方開始，並保留現有的程式碼與 git 歷程。
