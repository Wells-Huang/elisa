# 技能 (Skills)

技能是可重複使用的提示詞片段，能為您的 AI 代理提供比積木更詳盡的指令。

## 技能的作用

像是「它應該要能... 播放音效」這樣的積木是一項簡短的指令。而「技能」可以讓您提供更多細節：「使用 Web Audio API。在頁面載入時預載所有音效。當玩家收集到物品時播放硬幣音效。音量保持在 0.7...」。

透過將「使用技能」積木拖曳到畫布上，即可將技能附加到您的專案。

## 建立技能

1. 點擊側欄的 **技能 (Skills)** 按鈕。
2. 點擊 **+ 新增技能 (New Skill)**。
3. 輸入 **名稱**（例如：「像素藝術風格」）。
4. 選擇 **類別**：
   - **代理行為 (Agent behavior)** -- 控制代理的溝通與工作方式。
   - **功能細節 (Feature details)** -- 深入描述特定功能的細節。
   - **風格細節 (Style details)** -- 設定視覺風格、顏色與美學。
   - **複合流程 (Composite flow)** -- 透過視覺化方式將多個步驟鏈接起來（見下文）。
5. 撰寫 **指令** -- 代理將會遵循的詳細提示詞。
6. 點擊 **完成 (Done)**。

## 編輯與刪除技能

在技能彈窗中，每個技能都會顯示其名稱、類別以及指令預覽。點擊 **編輯 (Edit)** 進行修改，或點擊 **刪除 (Delete)** 將其移除。

## 範本庫

技能彈窗中有一個 **範本 (Templates)** 標籤，提供您可以一鍵新增的預設技能：

| 範本名稱 | 類別 | 作用 |
|----------|----------|-------------|
| 解釋每一步 | 代理 | 代理會用簡單的文字解釋每個步驟 |
| 適合兒童的程式碼 | 代理 | 使用簡單命名、大量註釋，不使用花俏技巧 |
| 詳細遊戲機制 | 功能 | 包含物理、碰撞、計分及難度進階流程 |
| 響應式佈局 | 功能 | 使用 flexbox/grid 的行動優先佈局 |
| 無障礙優先 | 功能 | 包含 ARIA 標籤、鍵盤導覽、語義化 HTML |
| 深色模式 | 風格 | 深色背景搭配鮮艷的強調色 |
| 像素藝術 | 風格 | 使用有限色票的復古 8 位元美學 |
| 簡報產生器 | 複合 | 詢問主題與格式，然後構建一份簡報 |
| 程式碼檢閱清單 | 複合 | 詢問關注點，然後執行針對性的檢閱 |

新增範本後，將「使用技能」積木拖到畫布上即可啟用它。

## 複合技能 (Composite skills)

複合技能使用視覺化流程編輯器將多個步驟鏈接起來。它們定義的是一個多步驟的工作流，而非單一的提示詞。

### 開啟流程編輯器

1. 建立一個新技能，並將類別設定為 **複合流程 (Composite flow)**。
2. 給予名稱，亦可視需求加入說明。
3. 點擊 **開啟流程編輯器 (Open Flow Editor)**。

### 流程積木

流程編輯器有自己的一套積木：

| 積木 | 作用 |
|-------|-------------|
| **技能流程 (Skill Flow)** | 入口點。必須是每個流程中的第一個積木。 |
| **詢問使用者 (Ask User)** | 暫停並向使用者提出包含選項的問題。將答案存儲在上下文變數中。欄位：問題、標題、逗點分隔的選項、儲存鍵 (store-as key)。 |
| **If** | 根據上下文的值進行分支。僅當值匹配時才執行內含積木。無 else 分支 —— 每種情況請使用多個 If 積木。 |
| **執行技能 (Run Skill)** | 調用另一個技能並儲存其產出。支援最高 10 層的巢狀調用並具備循環偵測。 |
| **執行代理 (Run Agent)** | 帶著提示詞範本產生 Claude 代理並儲存結果。 |
| **設定上下文 (Set Context)** | 將上下文變數設定為特定值。對組合或轉換值非常有用。 |
| **輸出 (Output)** | 產生流程的最終產出。這是最後一個積木（下方無接口）。 |

### 上下文變數

在任何文字欄位中使用 `{{key}}` 語法來引用先前步驟存儲的值：

```
詢問使用者 --> 存儲為 "topic"
執行代理 --> 提示詞 "構建一個 {{topic}} 應用程式" --> 存儲為 "result"
輸出 --> 範本 "已完成：{{result}}"
```

### 分支

If 積木沒有 else 分支。要處理多種情況，請鏈接多個 If 積木：

```
If answer 等於 "遊戲"    --> [執行代理：構建一個遊戲]
If answer 等於 "網站"    --> [執行代理：構建一個網站]
```

每個 If 積木會獨立評估，以第一個匹配項優先。

> **試試看**：建立一個名為「專案啟始員」的複合技能。新增一個「詢問使用者」積木，詢問「什麼類型的專案？」，選項為「遊戲, 網站, 工具」。針對每個選項分別新增「If」積木，並各包含一個提供具體指令的「執行代理」積木。最後在末端新增「輸出」積木。
